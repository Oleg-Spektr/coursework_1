# Приложение для анализа банковских транзакций 📊

Курсовая работа по разработке консольного приложения для фильтрации, поиска и аналитики финансовых операций из Excel-файлов. Проект генерирует JSON-данные для веб-страниц, предоставляет сервисы поиска и формирует аналитические Excel-отчеты.

## 🚀 Функционал приложения

### 1. Веб-страницы (`src/views.py`)
* **Главная страница (`main_page_view`)**: Генерирует JSON-ответ со сводной информацией за текущий месяц относительно переданной даты. 
  * Автоматически формирует приветствие в зависимости от времени суток (утро/день/вечер/ночь).
  * Группирует расходы по банковским картам и рассчитывает потенциальный кэшбэк.
  * Выводит топ-5 самых крупных транзакций, отсортированных по убыванию суммы.
  * Интегрируется со сторонними внешними API для получения актуальных курсов валют (USD, EUR) и стоимости акций из файла настроек пользователя.

### 2. Умные сервисы (`src/services.py`)
Реализованы с активным применением элементов функционального программирования (`filter`, `lambda`) и регулярных выражений (`re`):
* **Простой поиск (`simple_search`)**: Находит транзакции по подстроке в описании или категории. Поиск полностью нечувствителен к регистру букв.
* **Поиск по телефонным номерам (`search_by_phone_number`)**: Выделяет из общего массива операции, содержащие в описании мобильные телефоны различных форматов (например, `+7 (900) 000-00-00`, `89000000000`).
* **Поиск переводов физлицам (`search_transfers_to_individuals`)**: Находит только те переводы, в описании которых содержатся имена физических лиц с инициалами (например, `Валерий А.`).

### 3. Аналитические отчеты (`src/reports.py`)
* **Декоратор `@report_to_file`**: Кастомный декоратор, который перехватывает возвращаемый функциями `DataFrame` и автоматически записывает данные в Excel-файлы в папку `reports/`. Поддерживает гибкую передачу имени файла.
* **Траты по категории (`spending_by_category`)**: Формирует отчет по расходам в заданной категории за последние 3 месяца от указанной даты.
* **Траты по дням недели (`spending_by_weekday`)**: Рассчитывает средний чек расходов пользователя для каждого дня недели за последние 3 месяца.

## 🛠️ Установка и настройка проекта

Проект использует менеджер зависимостей **Poetry** для изоляции окружения и управления библиотеками (`pandas`, `requests`, `openpyxl`, `xlrd`).

1. **Клонируйте репозиторий и перейдите в его папку:**
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd coursework_1
   ```

2. **Настройте Poetry и установите все зависимости (включая инструменты разработчика):**
   ```bash
   poetry lock
   poetry install
   ```

3. **Подготовьте файлы данных:**
   * Убедитесь, что ваш исходный банковский файл `operations.xls` (или `xlsx`) размещен в папке `data/`.
   * В корне проекта должен находиться конфигурационный файл `user_settings.json` со списками отслеживаемых валют и акций.

## 💻 Запуск проекта

Для запуска демонстрационного скрипта, который обрабатывает ваш реальный Excel-файл, генерирует JSON-ответы в консоль и создает отчеты в папке `reports/`, выполните профессиональную команду запуска модуля:

```bash
poetry run python -m src.main
```

## 🧪 Тестирование и качество кода

Проект разработан по строгим промышленным стандартам качества:
* **Тесты**: Покрыт автоматическими юнит-тестами с использованием фикстур и моков в `pytest`. Для запуска тестов выполните:
  ```bash
  poetry run pytest
  ```
* **Линтинг и форматирование**: Код приведен к полному соответствию стандарту PEP 8 с помощью утилит `black`, `isort` и проверен линтером `flake8`:
  ```bash
  poetry run isort src/ tests/
  poetry run black src/ tests/
  poetry run flake8 src/ tests/
  ```
